Welcome to FaceLog

AI-Based Face Recognition Attendance Tracker

FaceLog is an intelligent attendance management system powered by artificial intelligence and facial recognition technology. Designed to automate the traditional attendance process, FaceLog accurately detects and verifies student identities using face recognition, eliminating the need for manual entry or ID cards. With seamless real-time detection, secure data storage in MongoDB, and a responsive dashboard for insights, FaceLog is a modern solution for educational institutions seeking efficiency, reliability, and smart analytics in classroom management. Built using Python, OpenCV, and Flask, FaceLog leverages state-of-the-art AI models to detect and match faces against a database of enrolled students. The system supports live webcam capture, batch face encoding, and automated CSV report generation, making it ideal for both small and large classroom setups. The user-friendly web interface allows administrators and faculty to monitor attendance records, view student details, and generate attendance reports on demand. Each face scan is logged with a timestamp and subject, ensuring traceability and eliminating proxy attendance. FaceLog is also highly customizable and scalable.

About FaceLog

The FacLog Attendance System is a smart and efficient facial recognition-based solution designed to automate the process of student attendance. Traditional methods of roll calls or manual sign-ins are time-consuming and prone to errors or even misuse. FacLog addresses these challenges by leveraging modern computer vision technology and artificial intelligence to recognize student faces in real time using a webcam, and accurately log their attendance.

FacLog allows each student to be registered with a photo and a unique registration number. During a session, the system captures faces from the live camera feed and compares them against the registered database. If a match is found, it marks the student as present for that subject and records the time and date. All attendance information is stored securely in a MongoDB database, and a CSV file is also generated for administrative convenience. The project includes a subject-wise tracking system, meaning attendance is managed independently for each course. Additionally, the student database keeps count of how many sessions each student has attended per subject, while also recording the total number of sessions conducted. This makes it easy for faculty and administrators to monitor both overall and individual participation. FacLog offers a simple yet powerful web interface built using Flask. Users can navigate through options such as recognizing students, adding new faces, and viewing records. The interface is user-friendly and designed to minimize manual intervention while maintaining accuracy and transparency.
